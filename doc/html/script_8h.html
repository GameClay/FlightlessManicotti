KL_API int kl_script_event_endframe ( kl_script_context_t  context,
kl_script_event_fence_t *  fence 
)

End a frame of script-events.

Script events are processed in frames. Calling this function on a script-context signifies that there are no more events for this frame. A fence parameter may be specified in order to allow the caller to wait on the fence.

Parameters:
contextThe script-context on which to push an end-of-frame event.
fenceOptional event-fence on which the caller can wait, or NULL if this functionality is not necesassary.
Returns:
KL_SUCCESS if successful. KL_ERROR if unsuccessful.

KL_API int kl_script_init ( kl_script_context_t *  context,
KL_BOOL  threaded,
size_t  event_queue_max 
)

Allocate and initialize a script-context.

This function may be called to allocate additional script-context. kl_initialize() creates a script context which should be used for most cases.

A script-context may run threaded, or non-threaded. If the context is allocated without threading, kl_script_event_pump() must be called to process a frame of script.

See also:
kl_script_is_threaded
Parameters:
contextThis holds the allocated script context.
threadedShould this script-context run in a thread.
event_queue_maxThe maximum number of events that can fit in the event queue for this script context.
Returns:
KL_SUCCESS if initialization was successful. KL_ERROR if initialization failed.
